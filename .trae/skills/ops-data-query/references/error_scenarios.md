# 通用错误处理方案（所有子技能共用）

> 本文件定义统一的错误处理逻辑。子技能 SKILL.md 中引用本文件即可。
> 所有子技能使用相同编号的错误场景，便于维护。

---

## 错误分类

| 严重度 | 标识 | 含义 | 处理策略 |
|--------|------|------|---------|
| FATAL | 🔴 | API 完全不可用、参数严重错误 | 立即降级到 Mock 数据 |
| WARN | 🟡 | 部分数据缺失、响应格式异常 | 返回可用数据，标注异常字段 |
| INFO | ℹ️ | 用户输入不完整、查询条件为空 | 引导用户提供更多信息 |

---

## 错误场景清单

### 场景 1：缺少必要参数

**严重度**：INFO（引导型错误）

**触发条件**：用户查询未提供必要筛选条件

**处理逻辑**：
1. 识别缺失的必填参数
2. 列出可选值供用户选择
3. 提供 1-2 个常用查询示例

**输出模板**：
```
我需要更多信息来完成查询。

{specific_options}

您也可以直接说，例如："查询贵州机房的{default_query}"
```

**各子技能具体引导内容**：

| 技能 | 缺失时引导 |
|------|---------|
| cmdb-server-query | 可选机房：贵州、北京、上海、广州、朝阳 |
| server-public-ip-query | 可选：按机房查询（如"贵州机房"），或提供内网IP查询 |
| project-deployment-query | 可选：按项目名查询，或按环境/状态筛选 |
| product-query | 可选：按产品名称模糊搜索，或查询所有启用的产品 |
| project-basis-query | 可选：按项目名称搜索，或按产品名称筛选 |

---

### 场景 2：参数格式错误

**严重度**：WARN

**触发条件**：用户输入的格式不符合规范

**常见错误类型**：

| 错误类型 | 用户输入示例 | 正确格式示例 |
|---------|------------|-------------|
| IP 地址格式错误 | "192.168.7" | "192.168.7.101" |
| 日期格式错误 | "2026-13-01" | "2026-05-10" |
| 枚举值超出范围 | "状态=9" | "状态=0~6" |

**输出模板**：
```
## 查询失败

**错误类型**：参数格式错误

**错误原因**：{error_reason}

**建议**：{correct_format_example}
```

---

### 场景 3：API 调用超时

**严重度**：FATAL（自动降级）

**触发条件**：HTTP 请求超时或连接失败

**处理逻辑**：
1. 立即切换到 Mock 数据
2. 在输出中明确标注"模拟数据"
3. 不提示用户重试（自动处理）

**输出模板**：
```
## 查询结果

> ⚠️ API 服务暂不可用，已切换为模拟数据（仅供演示）

{...标准表格...}

**说明**：当前显示的是模拟数据，请稍后重新查询获取真实数据。
```

---

### 场景 4：API 返回异常状态码

**严重度**：WARN / FATAL

| code | 含义 | 严重度 | 处理 |
|------|------|--------|------|
| 200 | 成功 | - | 正常处理 |
| 401 | 未授权 | FATAL | 降级，提示联系管理员 |
| 403 | 无权限 | FATAL | 降级，提示联系管理员 |
| 404 | 接口不存在 | FATAL | 降级，提示接口配置错误 |
| 500 | 服务器内部错误 | FATAL | 降级，提示稍后重试 |
| 其他 | 未知错误 | WARN | 降级，显示原始错误信息 |

**401/403 处理**：
```
## 查询失败

**错误类型**：权限不足

**错误原因**：当前账号无权访问此数据

**建议**：请联系管理员开通 CMDB 数据访问权限
```

**500 处理**：
```
## 查询失败

**错误类型**：服务异常

**错误原因**：CMDB 服务内部错误

**建议**：请稍后重试，或联系管理员检查服务状态
```

---

### 场景 5：无查询结果

**严重度**：INFO

**触发条件**：API 返回 `total = 0` 或 `records = []`

**处理逻辑**：
1. 返回简洁的空结果
2. 提供可用筛选项列表
3. 给出修正建议

**输出模板**：
```
## 查询结果

**查询条件**：{user_query}
**匹配技能**：{skill_name}

---

**结果摘要**：未查询到符合条件的记录

---

**说明**：{empty_note}

可用筛选项：
{available_options}

💡 您是否想说："查询{default_example}"？
```

---

### 场景 6：部分字段解析失败

**严重度**：WARN

**触发条件**：API 响应中某些字段缺失或类型错误

**处理逻辑**：
1. 使用空值占位符（`-`）填充缺失字段
2. 正常返回可用数据
3. 不中断流程

**示例**：
```
| 主机名 | IP地址 | 带宽 | 计费类型 |
|--------|--------|------|----------|
| gz-server-01 | 192.168.7.101 | 100Mbps | 按带宽 |
| gz-server-02 | 192.168.7.102 | - | - |
```

## Mock 数据降级策略

### 何时降级

| 触发条件 | 是否降级 | 备注 |
|---------|---------|------|
| HTTP 超时 | ✅ | 自动降级 |
| HTTP 500 | ✅ | 自动降级 |
| HTTP 401/403 | ✅ | 自动降级 |
| HTTP 404 | ✅ | 自动降级 |
| code ≠ 200 | ✅ | 自动降级 |
| 无网络连接 | ✅ | 自动降级 |
| 数据为空 | ❌ | 正常返回空结果 |

### 降级流程

```
API 请求
    ↓
请求成功？
  ├── 否 → 使用 Mock 数据
  │         ↓
  │      返回降级输出
  │         ↓
  │      在顶部标注 ⚠️
  │
  └── 是 → code == 200？
              ├── 是 → 正常处理响应
              └── 否 → 使用 Mock 数据 + 顶部标注 ⚠️
```

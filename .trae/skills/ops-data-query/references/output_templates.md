# 输出模板库（所有子技能共用）

> 本文件是唯一输出模板源。各子技能 SKILL.md 通过引用本文件获得标准输出格式。
> 每次修改只需改这一处，5 个子技能同步生效。

---

## 模板使用说明

| 模板类型 | 使用场景 | 变量占位符 |
|---------|---------|-----------|
| 标准输出 | 默认展示（前 10 条） | `{skill_name}`, `{user_query}`, `{condition}`, `{total}`, `{show_count}`, `{table_header}`, `{table_rows}` |
| 详细输出 | 用户要求"查看详细信息" | 同上，增加 `{detail_table_header}`, `{detail_table_rows}` |
| 降级输出 | API 调用失败时 | `{skill_name}`, `{user_query}`, `{params}`, `{total}`, `{table_header}`, `{table_rows}` |
| 错误输出 | 查询失败时 | `{error_type}`, `{error_reason}`, `{suggestion}` |

---

```
**查询范围**：{condition}

---

**结果摘要**：共查询到 {total} 条记录，当前显示前 {show_count} 条

{table_header}

{table_rows}

```

## 4. 错误输出

### 4.1 缺少必要参数

```
我需要更多信息来完成查询。

{specific_guidance}

您也可以直接补充说，例如："查询{default_location}机房的{query_type}"
```

### 4.2 参数格式错误

```
## 查询失败

**错误类型**：参数格式错误

**错误原因**：{error_reason}

**建议**：{suggestion}
```

### 4.3 API 调用超时

```
## 查询失败

**错误类型**：网络超时

**错误原因**：CMDB 服务响应超时，请稍后重试

**建议**：
1. 检查网络连接
2. 稍后重新查询
3. 如问题持续，请联系管理员
```

### 4.4 无查询结果

```
## 查询结果

**查询条件**：{user_query}
**匹配技能**：{skill_name}
**查询参数**：{params}

---

**结果摘要**：未查询到符合条件的记录

---

**说明**：{empty_state_note}
```

## 5. 字段表格生成规则

### 5.1 表格列数限制

- 标准模式：最多 5 列（避免过宽）
- 详细模式：最多 10 列（完整信息）

### 5.2 状态值格式化

| 原始值 | 格式化显示 |
|-------|----------|
| state=0 | 🟢 在线 |
| state=1 | 🟡 库存 |
| state=2 | ⏳ 计划上线 |
| state=3 | 🔧 维修中 |
| state=4 | ⚫ 已报废 |
| state=5 | ⬜ 待分配 |
| state=6 | 🔴 待清退 |
| status=0 | ✅ 成功 |
| status=1 | ❌ 失败 |
| status=2 | 🔄 进行中 |
| status=3 | ⏳ 待部署 |

### 5.3 时间格式化

- 完整时间：`YYYY-MM-DD HH:mm:ss`
- 简洁时间：`MM-DD HH:mm`
- 耗时（秒→中文）：`330秒 → 5分30秒`，`90秒 → 1分30秒`，`3600秒 → 1小时`

### 5.4 列表字段格式化

- `serverList`: `prod-guizhou-app-01(192.168.7.101) ✅ / prod-guizhou-app-02(192.168.7.102) ✅`
- `errorMessage`: `❌ 服务启动超时`
- 空字符串: `-`

## 6. 数量词识别与 pageSize 策略

| 用户输入 | pageSize（cmdb） | pageSize（其他） |
|---------|----------------|----------------|
| 一台/一个 | 1 | 1 |
| 几台/一些/几个 | 15 | 40 |
| 所有/全部 | 100 | 100 |
| 最新的/最近的 | 10 | 10 |
| 默认值 | 15 | 40 |

### 分页指令

| 用户输入 | 操作 |
|---------|------|
| 下一页 / 继续 / 更多 | `currentPage + 1` |
| 上一页 / 返回 | `currentPage - 1`（最小 1） |
| 第 N 页 | `currentPage = N` |
| 查看全部 | `pageSize = 100` |

## 7. 通用引导话术

### 无结果时的智能引导

```
可用筛选项：
- 机房：贵州、北京、上海、广州、朝阳
- 状态：在线、库存、维修中、已报废
- 类型：物理机、虚拟机、云服务器

💡 您是否想说："查询贵州机房的服务器"？
```

### 缺少机房参数时的引导

```
我需要更多信息来完成查询。

请问您想查询哪个机房的服务器？
可选机房：贵州、北京、上海、广州、朝阳

或者您可以提供具体的筛选条件，如：
- "查询贵州机房的在线物理机"
- "找一台贵州机房的测试服务器"
```
